**Markdown** es una forma sencilla de escribir texto estructurado usando una sintaxis muy ligera.

Fue creado por John Gruber en 2004 con una idea muy clara: que escribir contenido para la web fuese más fácil que escribir HTML a mano.

Hoy sigue siendo muy útil para blogs, documentación, notas, README de GitHub, Obsidian, Notion, Jekyll, Hugo, Astro y muchas herramientas de IA.

## Por qué usar Markdown

Markdown tiene varias ventajas:

- Es fácil de leer incluso sin renderizar.
- No te obliga a escribir HTML para cada formato.
- Funciona bien con Git.
- Es portable entre muchas herramientas.
- Es ideal para documentación y posts.
- Ayuda a separar contenido de diseño.

En este mismo blog, los posts viven como archivos Markdown dentro de Jekyll.

## Encabezados

Los encabezados se crean con `#`.

```markdown
# Título principal
## Sección
### Subsección
```

Normalmente el H1 lo gestiona el título del post, así que dentro del contenido conviene empezar por H2.

## Listas

Puedes crear listas con guiones:

```markdown
- Primer punto
- Segundo punto
- Tercer punto
```

O listas numeradas:

```markdown
1. Primer paso
2. Segundo paso
3. Tercer paso
```

## Formato básico

Para negrita y cursiva:

```markdown
**texto en negrita**
*texto en cursiva*
```

También puedes usar código inline:

```markdown
Usa `git status` para revisar cambios.
```

## Enlaces e imágenes

Un enlace se escribe así:

```markdown
[Texto del enlace](https://ejemplo.com)
```

Una imagen es parecido, pero con `!` delante:

```markdown
![Texto alternativo de la imagen](/images/ejemplo.jpg)
```

Aquí conviene recordar la importancia del [atributo alt en las imágenes](/blog/Atributo-ALT/), porque no todo es visual: también hay accesibilidad y contexto.

## Citas y bloques de código

Las citas usan `>`:

```markdown
> Esto es una cita.
```

Los bloques de código usan tres backticks:

````markdown
```html
<h1>Hola</h1>
```
````

## Markdown e IA

Markdown también se ha vuelto muy útil con herramientas de IA.

Los modelos entienden bastante bien estructuras con encabezados, listas, tablas y bloques de código. Por eso muchos prompts, planes, documentos técnicos y exports se escriben en Markdown.

También encaja muy bien con flujos de notas y conocimiento, como expliqué al hablar de [Apple Journal](/blog/apple-journal-la-app-de-diario-digital-de-apple/) o [GTD](/blog/gtd-flujo-de-trabajo-y-productividad-sin-estres/).

## Conclusión

Markdown sigue siendo una de las mejores formas de escribir contenido estructurado sin complicarse.

Es simple, portable y suficientemente potente para blogs, documentación, notas y flujos de trabajo modernos.
