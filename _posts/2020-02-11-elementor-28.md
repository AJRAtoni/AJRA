Elementor 2.8 llegó con tres novedades destacadas: modo oscuro, nuevo selector de color y mejoras en el editor responsive.

Para quienes construyen webs en WordPress con Elementor, eran cambios pequeños pero muy prácticos.

## Nuevo selector de color

El selector de color se rediseñó para guardar colores favoritos y reutilizarlos en diferentes partes de la web.

También mejoró la organización: se podían reordenar colores arrastrando y soltar para mantener una paleta más clara.

Esto ayuda a mantener consistencia visual. Si cada sección usa un azul ligeramente distinto, la web empieza a parecer improvisada.

## Modo oscuro

Elementor añadió modo oscuro en la interfaz.

No cambia la web que ve el visitante, pero sí mejora la experiencia de quien edita durante muchas horas.

Además, se podía configurar manualmente o dejar que siguiera el modo del sistema operativo.

## Mejoras responsive

El editor responsive también recibió mejoras.

Ahora era más claro ocultar secciones en determinados dispositivos y ver elementos "silenciados" cuando estaban ocultos en una vista concreta.

También se mejoró el selector de dispositivos para revisar orientaciones vertical y horizontal.

Este punto es clave. Diseñar solo en escritorio ya no tiene sentido. La experiencia móvil afecta a usabilidad, conversión y SEO, como expliqué en [mobile friendly](/blog/google-mobile-friendly/).

## Elementor dentro de WordPress

Elementor siguió creciendo porque redujo la barrera de entrada para crear páginas visuales en WordPress.

Pero conviene usarlo con cabeza: rendimiento, estructura, mantenimiento y plugins siguen importando. Ya vimos la evolución de WordPress con [Gutenberg](/blog/gutenberg/) y [WordPress 5.0](/blog/wordpress-5/).
