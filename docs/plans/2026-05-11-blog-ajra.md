# Blog AJRA Implementation Plan

> Para Hermes/Codex: implementar y mantener el blog de `ajra.es` en GitHub Pages sin romper la web estática existente.

Goal: añadir `/blog/`, páginas de post, navegación entre posts, featured images y RSS.

Architecture: usar Jekyll nativo de GitHub Pages porque el repositorio ya es HTML/CSS estático y GitHub Pages lo construye sin añadir Astro/Node ni migrar toda la web. La home sigue siendo HTML plano; el blog vive en `_posts`, `_layouts`, `blog/index.html` y `feed.xml`.

Tech Stack: GitHub Pages + Jekyll + Markdown/Kramdown + CSS existente.

## Decisión tecnológica

- Jekyll gana frente a Astro para este repo porque no requiere pipeline nuevo, npm, GitHub Actions ni migración de la home.
- Astro tendría sentido si más adelante quieres componentes, colecciones tipadas, búsqueda, tags avanzados o un rediseño completo.
- Para ahora: Jekyll = menos fricción, URLs limpias y RSS fácil.

## Tareas implementadas

1. Crear `_config.yml` con permalink `/blog/:title/`.
2. Crear `_layouts/default.html` y `_layouts/post.html`.
3. Crear `blog/index.html` como índice de posts.
4. Crear `feed.xml` RSS manual con Liquid.
5. Importar posts antiguos/locales desde `AJRAos/projects/recursos-by-ajra/PUBLISHED` hacia `_posts`.
6. Asignar featured image usando `images/posts/Newsletter-XXX.webp` cuando existe.
7. Añadir estilos de blog en `css/style.css`.
8. Añadir enlace a Blog/RSS desde la home.

## Siguientes mejoras opcionales

- Importar export completo de Substack/WordPress si AJRA lo proporciona.
- Añadir tags/categorías navegables.
- Añadir búsqueda local.
- Añadir plantilla de nuevo post en `_drafts`.
- Pasar a Astro solo si el blog crece y se quiere rediseño de todo el sitio.

## Verificación

- `bundle exec jekyll build` si Ruby/Jekyll está instalado.
- Alternativa rápida: validar que existen `_site/blog/index.html`, `_site/feed.xml` y posts bajo `_site/blog/...` tras build de GitHub Pages.
